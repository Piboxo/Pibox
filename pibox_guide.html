Guide de setup
PiBox

Votre PiBox est pré-configuré — la plupart des réglages sont déjà faits. Ce guide vous explique ce qui a été mis en place, ce que vous pouvez vérifier, personnaliser, et les quelques actions que vous devez faire vous-même (création de comptes, configuration routeur).

✓ Déjà configuré sur votre PiBox BIOS optimisé (virtualisation ON, démarrage rapide, no sleep) · Windows mode haute performance · Apps en démarrage automatique · Priorités processus réglées · Reboot automatique dimanche 23h · Pare-feu ajusté pour les apps
→ Ce que vous devez faire Créer vos comptes sur chaque app (avec votre email) · Configurer votre routeur (IP fixe + UPnP) · Connecter vos wallets/paiements sur chaque plateforme
Étape 1

Créer vos comptes

Chaque app nécessite un compte indépendant. Vous gérez vos gains directement sur leurs plateformes — PiBox n'a pas accès à vos comptes.

Salad.io earn.app Honeygain MystNodes bitping
Requis
Créer un compte sur chaque application

Ouvrez chaque application sur votre PiBox. Elles sont déjà installées et se trouvent dans le bureau ou la barre des tâches. Créez un compte via Sign Up avec votre email.

Une fois connecté, l'app commence automatiquement à gagner. Aucune configuration supplémentaire n'est nécessaire dans l'app elle-même.

Étape 2

Email recommandé : ProtonMail

Utiliser Gmail ou un email courant pour ces apps n'est pas recommandé pour deux raisons : confidentialité et détection de bots.

Recommandé
Créer un compte ProtonMail proton.me — gratuit

Gmail traque l'activité et peut signaler votre compte si des connexions automatiques répétées sont détectées (ce que font les apps de revenus passifs en se connectant à leurs serveurs). Certaines plateformes ont des politiques anti-bot qui peuvent suspendre des comptes Gmail.

ProtonMail est chiffré de bout en bout, basé en Suisse, sans tracking publicitaire. Il est considéré comme plus sérieux par les plateformes DePIN/bandwidth.

→ proton.me Gratuit, sans pub, sans tracking. Créez un compte dédié à vos apps PiBox (ex: votrenom.pibox@proton.me).
Étape 3 — Réseau

IP fixe locale pour votre PiBox

Les apps de bande passante fonctionnent mieux avec une IP locale stable. Configurer une IP fixe via votre routeur évite les déconnexions quand le bail DHCP expire.

Recommandé
Réserver une IP fixe dans votre routeur (bail DHCP statique)

Cette opération se fait dans l'interface de votre box/routeur — elle est différente selon l'opérateur mais le principe est identique.

1. Trouver l'adresse MAC de votre PiBox :

// Dans l'invite de commandes Windows (Win+R → cmd) : ipconfig /all // Notez "Adresse physique" de votre carte réseau Ethernet // Format : XX-XX-XX-XX-XX-XX

2. Dans votre box : Allez dans Réseau → DHCP → Baux statiques (ou similaire). Associez l'adresse MAC de votre PiBox à une IP fixe de votre choix, par exemple 192.168.1.50.

Opérateurs courants :
• Livebox Orange : 192.168.1.1 → Réseau → Serveur DHCP
• Freebox : 192.168.1.254 → Paramètres DHCP
• SFR Box : 192.168.1.1 → Réseau local → DHCP
• Bbox Bouygues : 192.168.1.254 → Réseau local

Étape 4 — Réseau

UPnP & ouverture de ports

MystNodes notamment nécessite une connectivité entrante pour fonctionner en tant que nœud VPN résidentiel et être mieux rémunéré.

Recommandé
Activer UPnP sur votre routeur

UPnP (Universal Plug and Play) permet aux apps de demander automatiquement l'ouverture des ports dont elles ont besoin. C'est l'option la plus simple.

Dans l'interface de votre box, cherchez UPnP ou NAT/PAT et activez-le. Sur la plupart des boxes françaises il est déjà activé par défaut.

Optionnel
Ouvrir manuellement le port MystNodes (si UPnP inactif)

Si UPnP est désactivé et que vous souhaitez maximiser les revenus MystNodes, ouvrez le port 4449 (UDP+TCP) en redirection vers l'IP fixe de votre PiBox.

Vérifier la connectivité MystNodes Dans l'app MystNodes → Settings → Node status. Si vous voyez "NAT type: Full cone" ou un score élevé, votre connectivité est bonne. Si "Symmetric NAT", l'UPnP ou l'ouverture de port améliorera vos gains.
Étape 5 — Windows

Applications au démarrage automatique

Déjà configuré sur votre PiBox. Voici comment vérifier et modifier si besoin.

Pré-configuré
Vérifier les apps en démarrage automatique

Ouvrez le Gestionnaire des tâches : Ctrl+Shift+Échap → onglet Démarrage.

Vous devriez voir toutes les apps PiBox avec le statut Activé. Si une app est sur "Désactivé", clic droit → Activer.

Apps qui doivent être "Activé" Salad · earn.app · Honeygain · MystNode Launcher · bitping
Étape 6 — Windows

Priorité des processus

Les apps sont configurées en priorité "Au-dessus de la normale" pour qu'elles ne soient pas throttlées par Windows lors d'une activité sur le PC.

Pré-configuré
Vérifier / modifier la priorité d'un processus

Gestionnaire des tâches → onglet Détails. Cherchez le processus de l'app (ex: salad.exe). Clic droit → Définir la priorité.

La priorité recommandée est Supérieure à la normale. Évitez "Temps réel" — cela peut déstabiliser Windows.

Note La priorité se remet à "Normal" au redémarrage. C'est pourquoi un script de démarrage est inclus sur votre PiBox pour réappliquer automatiquement les priorités au boot.
Étape 7 — Windows

Veille désactivée — vérification

La mise en veille est désactivée au niveau BIOS et Windows. Voici comment vérifier.

Pré-configuré
Confirmer que la veille est bien désactivée

Panneau de configuration → Options d'alimentation → Modifier les paramètres du mode → Mettre l'ordinateur en veille : Jamais

Ou via PowerShell :

# Vérifier le plan actuel powercfg /query SCHEME_CURRENT SUB_SLEEP STANDBYIDLE # La valeur doit être 0x00000000 (jamais) # Si besoin de forcer : powercfg /change standby-timeout-ac 0 powercfg /change monitor-timeout-ac 0
Étape 8 — Windows

Reboot automatique hebdomadaire

Un redémarrage est planifié chaque dimanche à 23h. Cela nettoie la mémoire, applique les mises à jour Windows et maintient les apps en bonne santé.

Pré-configuré
Voir la tâche planifiée (Planificateur de tâches)

Ouvrez le Planificateur de tâches : Win+R → taskschd.msc → Bibliothèque → cherchez PiBox_WeeklyReboot.

Vous y verrez le planning configuré (dimanche 23h00).

Optionnel
Changer l'heure ou le jour du reboot

Dans le Planificateur de tâches, double-cliquez sur PiBox_WeeklyReboot → onglet Déclencheurs → double-cliquez sur le déclencheur → modifiez le jour et l'heure → OK.

Recommandation Choisissez une heure creuse où votre connexion est peu utilisée. 23h en semaine est idéal. Évitez les heures de pointe (18h–22h) — le reboot prend ~2 minutes pendant lesquelles les apps sont hors ligne.
Optionnel
Désactiver le reboot automatique

Dans le Planificateur de tâches, clic droit sur PiBox_WeeklyReboot → Désactiver. Ou supprimer complètement avec Suppr.

Attention Sans reboot hebdomadaire, Windows peut accumuler des mises à jour en attente et redémarrer au moment le plus inopportun. Nous recommandons de garder la tâche active ou de planifier vos propres reboots manuels.
Optionnel
Créer vous-même la tâche (si supprimée)
# PowerShell en administrateur — crée un reboot chaque dimanche 23h00 $action = New-ScheduledTaskAction -Execute "shutdown.exe" -Argument "/r /f /t 60" $trigger = New-ScheduledTaskTrigger -Weekly -DaysOfWeek Sunday -At "23:00" $settings = New-ScheduledTaskSettingsSet -ExecutionTimeLimit (New-TimeSpan -Minutes 5) Register-ScheduledTask -TaskName "PiBox_WeeklyReboot" -Action $action -Trigger $trigger -Settings $settings -RunLevel Highest -Force

Adaptez Sunday (lundi=Monday, mardi=Tuesday…) et 23:00 selon vos préférences.

Information

Configuration BIOS (pré-faite)

Ces réglages ont été effectués avant expédition. Vous n'avez pas besoin d'entrer dans le BIOS sauf en cas de réinitialisation.

Pré-configuré
Ce qui a été configuré dans le BIOS
  • Virtualisation Intel VT-x activée — nécessaire pour certaines apps (MystNodes notamment)
  • Wake on LAN activé — permet le redémarrage à distance si besoin
  • Power On After Power Loss réglé sur "Power On" — la PiBox redémarre automatiquement après une coupure de courant
  • Fast Boot activé — démarrage plus rapide
  • Deep Sleep / S3/S4 désactivé — empêche la mise en veille profonde au niveau matériel
  • Démarrage sur secteur — pas de bouton power nécessaire après coupure EDF
Pour accéder au BIOS si besoin Appuyez sur F1 ou Entrée au démarrage du Lenovo (logo noir Lenovo). Mot de passe BIOS : non configuré (sauf mention contraire dans votre colis).

Étape 3 — Wallet crypto

Créer un wallet MetaMask

Plusieurs apps (MystNodes, bitping, Traffmonetizer) paient en crypto. Un wallet MetaMask gratuit vous permet de recevoir ces paiements sans compte exchange.

Recommandé
Installer MetaMask metamask.io — gratuit

MetaMask est le wallet Web3 le plus répandu. Extension Chrome/Firefox/Edge ou app mobile.

1. Allez sur metamask.io (vérifiez l'URL — des faux sites existent).

2. "Download for Chrome" → Ajouter l'extension → Créer un nouveau wallet.

3. CRITIQUE : Notez votre seed phrase (12 mots) sur papier. Rangez-la en lieu sûr. Ne la saisissez jamais en ligne.

4. Votre adresse ressemble à : 0x1a2b3c...f9e8. C'est cette adresse que vous rentrez dans les apps pour recevoir vos crypto.

Ne partagez jamais votre seed phrase Ni avec PiBox, ni avec les apps, ni avec personne. Quiconque possède vos 12 mots contrôle votre wallet. Créez un wallet dédié PiBox — pas votre wallet principal.
Pas d'infos personnelles requises MetaMask ne demande ni nom, ni adresse, ni identité. Wallet anonyme, lié uniquement à votre seed phrase.
Optionnel
Ajouter le réseau Polygon — frais réduits

Certaines apps paient sur Polygon plutôt qu'Ethereum pour économiser les frais. MetaMask → Settings → Networks → Add Network → Add manually :

Network Name: Polygon RPC URL: https://polygon-rpc.com Chain ID: 137 Symbol: MATIC Explorer: https://polygonscan.com
Sécurité

Partage de réseau — ce que vous devez savoir

PiBox partage votre connexion internet. Voici ce qui se passe réellement, les garanties en place, et les bonnes pratiques.

À lire
Comment fonctionne le partage réseau

Les apps bandwidth (Salad, earn.app, Honeygain) transmettent du trafic chiffré via votre connexion pour le compte d'entreprises (vérification de prix, SEO, accès géolocalisé à des contenus publics).

MystNodes fait de même pour le routage VPN — des utilisateurs du réseau Mysterium accèdent à internet via votre IP résidentielle, de façon chiffrée.

Chiffrement assuré par les services externes Le trafic est chiffré de bout en bout par les plateformes. Vous ne routez pas de trafic en clair. Le contenu illégal est contractuellement interdit côté plateforme — PiBox ne porte pas de responsabilité sur l'usage fait de votre bande passante par les clients des plateformes.
Recommandation : réseau séparé si possible Si votre routeur supporte un réseau Wi-Fi invité ou VLAN, configurer PiBox dessus isole votre trafic personnel du trafic PiBox. Bonne pratique mais non obligatoire.
Important
Minimiser vos informations personnelles
  • Email dédié ProtonMail — pas votre email principal
  • Username neutre qui ne révèle pas votre identité réelle
  • Wallet MetaMask dédié PiBox — pas votre wallet principal avec des fonds
  • PayPal : seul endroit où vos infos personnelles sont requises — géré directement par PayPal
  • Ne connectez jamais vos wallets principaux aux apps PiBox
  • Ne partagez jamais votre seed phrase MetaMask, même avec le support d'une app

Applications

Apps recommandées & sources de données

La sélection d'applications installées sur votre PiBox est basée sur nos propres benchmarks mensuels et sur les données communautaires de sharedata.app.

Info
Comment nous sélectionnons les apps

Nous testons chaque application sur un nœud actif en France (IP résidentielle, même configuration que votre PiBox). Nous mesurons les revenus réels sur 30 jours minimum avant d'inclure une app dans la configuration.

Nos données sont croisées avec sharedata.app, une communauté indépendante qui agrège les gains réels de milliers de nœuds dans le monde.

Si une meilleure app émerge Nous publions les mises à jour de configuration sur notre canal (email/page web). Si le classement change significativement, vous serez notifié avec les instructions pour mettre à jour votre stack.
Optionnel
Apps à tester en supplément (non pré-installées)

Ces apps peuvent s'ajouter au stack existant. Testez-les une par une sur 2–3 semaines en surveillant l'impact sur les autres :

Repocket Traffmonetizer Peer2Profit

Ne jamais installer Repocket et Traffmonetizer simultanément — ils font le même travail et se cannibalisent.
